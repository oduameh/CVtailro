Business Model

Admin (owner) provides the OpenRouter API key. Users access the full service for free. Admin pays API costs. Rate limiting controls per-user consumption.

2. Pipeline

6-Stage Pipeline

StageAgentExecutionTypePurpose
1 JobIntelligenceAgent Parallel with Stage 2 LLM Extract structured job requirements, skills, seniority
2 ResumeParserAgent Parallel with Stage 1 LLM Parse resume into structured roles, skills, education
3 GapAnalysisAgent Sequential Pure Python Compare job vs resume: match score, missing keywords, TF-IDF cosine similarity
4 BulletOptimiserAgent Parallel per role LLM Rewrite bullets to inject keywords, strengthen verbs, add metrics
5 ResumeOptimiserAgent Parallel with Stage 6 LLM Produce unified optimised resume (ATS + recruiter friendly)
6 FinalAssemblyAgent Parallel with Stage 5 LLM Generate interview talking points from tailored resume

Concurrency Controls

ParameterValueLocation
MAX_CONCURRENT_PIPELINES5app/services/pipeline.py
MAX_QUEUE_DEPTH50app/services/pipeline.py
JOB_TTL_SECONDS900 (15 min)app/services/pipeline.py
Mechanismthreading.Semaphore for concurrency, counter for queue depth
ⓘ Bullet Optimiser splits into parallel threads when 3 or more roles are detected, reducing processing time from ~5 minutes to ~10 seconds.

6. Frontend

Main App Flow

#StepDetails
1Page loadFetch /api/status to check service config. Fetch /api/models if user model selection is enabled. Load saved resumes list.
2Model selectionDropdown populated from RECOMMENDED_MODELS. Admin can disable user model selection.
3Submit tailoringPOST /api/tailor with multipart form: resume PDF + job description text + selected model + rewrite mode.
4SSE progressEventSource(/api/progress/<job_id>) — receives stage names, percentages, status updates in real-time.
5Results displayOn complete event: fetch /api/result/<job_id>. Show match scores (before/after), optimised resume, talking points, section scores.
6DownloadsTemplate picker (8 PDFs) + DOCX download. Each links to /api/download/<job_id>/<filename>.

Admin Panel

Auth: Password login via /admin/api/login or auto-auth if logged-in user has is_admin=True.

Dashboard tabs:

  • Overview — Stats cards, recent jobs, live pipeline state
  • Users — User list with job counts, admin promotion
  • Analytics — Jobs over time chart, token usage, cost
  • Configuration — API key, default model, rate limits, model selection toggle
  • Diagnostics — Health checks (DB, OpenRouter, R2, Redis, pipeline)
  • Errors — Pipeline error log (in-memory, last 50)
  • Observability Hub — Product usage, reliability, cost, audit trail, alerts

Config save flow: Frontend POSTs JSON to /admin/api/config. API key is masked on read (sk-or-v1***). Masked values are ignored on write (only real keys are persisted).

Key testing: POST /admin/api/test-key hits https://openrouter.ai/api/v1/models with the provided key.

CSRF Double-Submit Pattern

CSRF is handled at the framework level via Flask-WTF, but admin_bp and api_bp are CSRF-exempt (session-based auth, no form tokens needed).

For non-exempt routes, the after_request handler sets a csrf_token cookie (httponly=False, SameSite=Lax). Frontend reads this cookie and sends it in the X-CSRFToken header.

Auto-recovery: If the session loses its CSRF token (e.g., session expiry or backend change), the cookie is automatically regenerated on the next response.

8. Troubleshooting

IssueCauseFix
CSRF token missing/invalid Session expired or cookie cleared. The csrf_token cookie was lost or the session token doesn't match. Reload the page — the after_request handler auto-regenerates the cookie when missing. For api_bp and admin_bp routes, CSRF is already exempt.
API key not working / "Service not configured" No OPENROUTER_API_KEY env var set, or DB key is empty/masked. Set the OPENROUTER_API_KEY environment variable in Railway. Use the admin panel's "Test Key" button to validate. Remember: env var always overrides DB.
Model returns empty/malformed JSON Free models often fail on structured JSON output. Model doesn't support the required output complexity. Switch to GPT-4o-mini (default, most reliable). Avoid free models for production. The BaseAgent has retry logic with JSON extraction/fix, but some models are fundamentally unreliable.
Download returns 404 / "Job not found" In-memory job expired (TTL 15 min), R2 file missing, and DB has no stored markdown to regenerate from. Check JOB_TTL_SECONDS (900s default). Ensure R2 is configured for persistent storage. The DB fallback regenerates PDFs from ats_resume_md — verify that column is populated.
Queue saturation / "Service is busy" More than 50 jobs queued (MAX_QUEUE_DEPTH) or all 5 semaphore slots in use. Wait for current jobs to complete. Increase MAX_CONCURRENT_PIPELINES or MAX_QUEUE_DEPTH in app/services/pipeline.py (requires restart). Monitor via /admin/api/live-stats.
Rate limited (429) User exceeded per-endpoint limits (e.g., 10 tailoring jobs/hr) or admin login attempts (5/min). Wait for the rate limit window to reset. Adjust limits in route decorators (@limiter.limit()). Admin login has a separate IP-based lockout (15 min after repeated failures).
DB connection error / "Could not connect to server" PostgreSQL unreachable, connection pool exhausted, or DATABASE_URL misconfigured. Check Railway database status. Verify DATABASE_URL uses postgresql:// prefix (auto-corrected from postgres://). Pool config: 20 connections + 30 overflow, 10s timeout, 5min recycle, pre-ping enabled.
Local dev: WeasyPrint / cairo errors Missing system libraries for WeasyPrint (cairo, pango, gdk-pixbuf). macOS: brew install cairo pango gdk-pixbuf libffi
Ubuntu: apt install libcairo2 libpango-1.0-0 libpangocairo-1.0-0 libgdk-pixbuf-2.0-0
The Dockerfile already installs these.
Local dev: Google OAuth redirect mismatch OAuth redirect URI doesn't match Google Cloud Console configuration. Add http://localhost:5050/auth/callback as an authorized redirect URI in the Google Cloud Console. Set GOOGLE_CLIENT_ID and GOOGLE_CLIENT_SECRET env vars.
Pipeline timeout / Gunicorn worker killed Complex resume + slow model exceeds Gunicorn's 300s timeout. The pipeline runs in a background thread and shouldn't be affected by Gunicorn timeout. If the SSE connection drops, the frontend should reconnect. Ensure Gunicorn --timeout 300 is set.
R2 upload fails silently R2 credentials missing or bucket doesn't exist. Pipeline continues without cloud storage. Set all R2 env vars (R2_ACCOUNT_ID, R2_ACCESS_KEY_ID, R2_SECRET_ACCESS_KEY, R2_BUCKET). Check /admin/api/diagnostics for R2 health status.
✓ Run /admin/api/diagnostics for a comprehensive health check covering database, OpenRouter, R2, Redis, and pipeline status.
